## 5. 开发规范

### 5.1 代码规范

**前端：**
- ESLint + Prettier
- TypeScript严格模式
- 组件命名：PascalCase
- 文件命名：kebab-case

**后端：**
- ESLint + Prettier
- TypeScript严格模式
- RESTful API规范
- 统一错误处理

### 5.2 Git工作流

**分支策略：**
```
main (生产)
  ├── develop (开发)
  │   ├── feature/customer-management
  │   ├── feature/funnel-system
  │   └── feature/dashboard
  └── hotfix/critical-bug
```

**提交规范：**
```
feat: 新功能
fix: Bug修复
docs: 文档更新
style: 代码格式
refactor: 重构
test: 测试
chore: 构建/工具
```

### 5.3 Code Review

- 所有代码必须经过Review
- 至少1人Approve才能合并
- 关键模块需要2人Approve
