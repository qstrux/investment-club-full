# 快速启动指南

## 🚀 5分钟了解项目

### 这是什么？

一个**企业级CRM系统**，专为**私域运营团队**设计，基于**三层漏斗增长模型**：

```
ToFu (引流层) → MoFu (培育层) → BoFu (转化层) → VIP客户
```

### 核心价值

| 问题 | 解决方案 |
|-----|---------|
| 新人培训慢（30-90天） | **7天上手**系统 |
| 数据统计靠Excel | **自动化**报表 |
| 客户管理混乱 | **360度**画像 |
| 凭经验决策 | **数据+AI**推荐 |
| 各自为战 | **标准化**流程 |

### 9大核心模块

1. **客户增长漏斗** - ToFu/MoFu/BoFu全流程管理
2. **CRM客户管理** - 360度画像 + 智能标签
3. **话术素材库** - 12大模型 + 智能推荐
4. **员工运营** - 任务管理 + KPI追踪
5. **培训系统** - 7/30/90天培训路径
6. **SOP引擎** - 智能推荐下一步动作
7. **数据仪表板** - 实时可视化
8. **风控预警** - 出金风险监控
9. **团队协作** - 权限管理 + 消息通知

## 🛠️ 立即开始开发

### 前置条件

- Node.js 20+
- PostgreSQL 14+
- Redis 7+
- Docker (可选)

### 第一步：环境准备

```bash
# 1. 克隆代码仓库（待创建）
git clone https://github.com/your-org/crm-system.git
cd crm-system

# 2. 安装依赖
cd backend
npm install

cd ../frontend
npm install

# 3. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入数据库连接等信息

# 4. 初始化数据库
npm run db:migrate

# 5. 启动开发服务器
npm run dev
```

### 第二步：查看效果

- 前端：http://localhost:3000
- 后端API：http://localhost:4000
- API文档：http://localhost:4000/api-docs

## 💡 常见问题

### Q1: 这个系统适合什么规模的团队？

**A**: 
- **最小**: 5-10人团队
- **推荐**: 20-50人团队
- **最大**: 支持500+用户

### Q2: 开发需要多少人？

**A**:
- **MVP**: 7人团队（前端2 + 后端2 + UI1 + 测试1 + PM1）
- **完整版**: 11人团队

### Q3: 总成本多少？

**A**:
- **人力成本**: ¥1,305,000 (90天)
- **基础设施**: ¥121,000/年
- **总计**: 约 ¥1,476,000

### Q4: 可以只开发部分功能吗？

**A**: 可以！建议按阶段开发：
1. 先做MVP（30天）验证价值
2. 再做标准版（60天）完善功能
3. 最后做企业版（90天）AI增强

### Q5: 技术栈可以更换吗？

**A**: 可以！文档提供了备选方案：
- 后端：Node.js 或 Java Spring Boot
- 前端：React 或 Vue
- 数据库：PostgreSQL 或 MySQL

### Q6: 支持哪些部署方式？

**A**:
- **开发/测试**: Docker Compose
- **生产环境**: Kubernetes
- **云服务**: AWS / 阿里云 / 腾讯云
